Software Security 2020/2021

This course unit introduces students to basic and advanced approaches to formally build verified trustworthy software systems, where trustworthy comprise five attributes: reliability, availability, safety, resilience and security.

This course will be taught by Lucas Cordeiro for post-graduate students in Computer Science. This page contains information about the course.

Overview

Software is subjected to numerous forms of attack such as memory corruption, buffer overflows and injection; these flaws are often too complex or expressive to be manually detected by the software developer.

Tools and techniques are used in order to prevent and detect software flaws, which are too complex to be manually found, e.g., modelling, code reviews, funzzing, static and dynamic code analyses, program verification and code tainting.

Relationship to Other Courses

Software Security involves people and practices, to build software systems to ensure confidentiality, integrity and availability. Therefore, this course has connections to other disciplines: cyber-Security, cryptography, automated reasoning and verification, logic and modelling, agile and test-driven development, software engineering concepts and systems governance.

Prerequisites

Basic programming skills, incl. familiarity with C and Python 3 In more detail:

  • For C, you should at least know how pointers and dynamic memories work.
  • For Python, you should know how to do basic algorithms and interact with the host system.

Basic notions in Linux sysadmin:

  • Create a web server.
  • Understand the difference between user space and kernel space.

Learning Outcomes

  • Define standard notions of security and use them to evaluate the system's confidentiality, integrity and availability
  • Explain standard softwre security problems in real-world applications
  • Use testing and verification techniques to reason about the system's safety and security
